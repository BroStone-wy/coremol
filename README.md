# CoReMol

CoReMol is a residual atom-pair communication adapter for molecular property prediction. The repository contains the core implementation, experiment configs, tests, and saved reports for AttentiveFP and Graphformer backbones.

The current code supports:

- Backbones: `attentivefp`, `graphformer`, `dmpnn`
- Tasks: MoleculeNet-style classification and regression datasets
- Variants: `base`, `coremol`
- Interpretability: training-time `TCM@k` diagnostics and post-hoc Full TCM via `scripts/compute_tcm_variants.py`

## Fresh Server Setup

```bash
git clone <github-url> coremol
cd coremol
conda create -n GNRF python=3.11 -y
conda activate GNRF
pip install -r requirements.txt
PYTHONPATH=. python scripts/prepare_datasets.py
PYTHONPATH=. pytest -q tests
```

See `ENVIRONMENT.md` for CUDA/PyTorch/PyG installation details.

## Run Examples

Classification:

```bash
PYTHONPATH=. python scripts/run_stage1_gate.py \
  --backbone attentivefp \
  --datasets BBBP \
  --seeds 0 1 2 \
  --variants base coremol
```

Graphformer regression:

```bash
PYTHONPATH=. python scripts/run_stage1_gate.py \
  --backbone graphformer \
  --datasets ESOL FREESOLV LIPO \
  --seeds 0 1 2 \
  --split_strategy random \
  --random_train_fraction 0.8 \
  --random_valid_fraction 0.1 \
  --normalize_regression \
  --variants base coremol
```

Full TCM:

```bash
PYTHONPATH=. python scripts/compute_tcm_variants.py \
  --run_dir <RUN_DIR> \
  --datasets <DATASET> \
  --seeds 0 1 2 \
  --backbone graphformer \
  --max_graphs 96
```

## Repository Policy

The repository intentionally excludes local datasets, checkpoints, and downloaded reference repositories:

- `data/` is regenerated by `scripts/prepare_datasets.py`.
- `results/` contains local training artifacts and is not committed.
- `third_party/` can be regenerated by `scripts/download_reference_repo.sh`.

See `RELEASE_MANIFEST.md` for the full include/exclude policy.
